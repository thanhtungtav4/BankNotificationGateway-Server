2. Chữ ký bảo mật HMAC

                Xác thực nguồn gốc các gói tin từ thiết bị và webhook
              

              
Để đảm bảo tính toàn vẹn và chống giả mạo, tất cả yêu cầu HTTP từ điện thoại gửi lên hoặc từ Webhook gửi đi đều được ký bằng thuật toán HMAC-SHA256 với khoá bí mật thiết bị (device_secret) hoặc webhook secret.

              
Cơ chế tạo chữ ký (Signature):

              
Chuỗi ký được tạo bằng cách nối chuỗi Timestamp (Unix Epoch) và Raw JSON Request Body ngăn cách bởi dấu chấm .. Chữ ký sau đó được băm bằng thuật toán SHA256 kèm theo Secret.

              
signature = HMAC-SHA256(timestamp + "." + raw_body, secret)

              
Các HTTP Header đi kèm yêu cầu:

              
    
                
  • X-Device-Id: ID của thiết bị gửi tin.
    • 
                
  • X-Timestamp: Thời gian gửi tin (Unix timestamp).
    • 
                
  • X-Signature: Chữ ký HMAC-SHA256 được tính toán dưới dạng chuỗi hexa.
    • 
              

              
              
Ví dụ code kiểm tra chữ ký (PHP):

              
<?php
$signature = $_SERVER['HTTP_X_SIGNATURE'] ?? '';
$timestamp = $_SERVER['HTTP_X_TIMESTAMP'] ?? '';
$body = file_get_contents('php://input');

$expectedSignature = hash_hmac('sha256', $timestamp . '.' . $body, $secretKey);
if (hash_equals($expectedSignature, $signature)) {
    // Chữ ký hợp lệ
}

4. Swagger Playgrounds (Thử nghiệm API Tương tác)

                Nhập dữ liệu và thực thi cuộc gọi trực tiếp từ trình duyệt
              

              

            


            
            

              

                
5. API dành cho Thiết bị di động (Mobile App)

              

              
Các endpoint này được sử dụng bởi ứng dụng Android Gateway để gửi dữ liệu về Server. Toàn bộ đều yêu cầu Chữ ký HMAC.

              

                
                  

                    Method & Route
                    Chức năng
                    Tham số body
                  

                
                
                  

                    POST /v1/mobile/pair
                    Kích hoạt ghép đôi điện thoại với Token sinh ra từ web dashboard.
                    pairing_token, device_name
                  

                  

                    POST /v1/mobile/heartbeat
                    Báo cáo trạng thái pin, trạng thái cắm sạc và số lượng tin nhắn trong hàng chờ mỗi 15 phút.
                    battery_level, is_charging, queue_pending
                  

                  

                    POST /v1/mobile/notifications
                    Gửi thông tin SMS / Notification của ngân hàng đã đọc được lên hệ thống.
                    package_name, app_name, title, text, posted_at, notification_key
                  

                
              

            


            
            

              

                
6. API Quản lý (Dashboard API)

              

              
Dashboard API yêu cầu token định dạng Bearer Token trong header Authorization.

              

                
                  

                    Method & Route
                    Chức năng
                  

                
                
                  

                    POST /admin/auth/login
                    Đăng nhập lấy Sanctum Token truy cập quản trị hệ thống.
                  

                  

                    GET /tenants
                    Danh sách các Tenants (Khách hàng) đang quản lý.
                  

                  

                    POST /tenant-webhooks
                    Thêm cấu hình Webhook URL mới cho Tenant.
                  

                  

                    POST /tenant-webhooks/{id}/parser-config
                    Lưu Regex bóc tách tuỳ chỉnh cho Webhook chỉ định.
                  

                  

                    GET /audit-logs
                    Truy xuất danh sách nhật ký kiểm toán hệ thống.
                  

                  

                    GET /v1/integration/transactions/lookup
                    Tra cứu giao dịch ngân hàng theo Mã đơn hàng (booking_id) và số tiền (amount).
